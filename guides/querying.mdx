The main purpose of CocoIndex is indexing — building derived data that enables efficient querying. You can use any library or framework to query that data. CocoIndex also provides seamless integration points between indexing and querying workflows:

- Share transformations (such as embedding computation) between indexing and query time.
- Register query handlers discoverable by tools like CocoInsight.
- Resolve default table names generated by CocoIndex naming conventions.

## Transform flows

When building a vector index, the embedding computation must be consistent between indexing and querying. Extract that shared logic into a *transform flow*: a standalone function that can be called both during indexing and at query time.

### Defining a transform flow

1. Extract the shared logic into a function that takes one or more `cocoindex.DataSlice[T]` inputs and returns a `cocoindex.DataSlice[T]` output. Annotate types explicitly.
2. Decorate it with `@cocoindex.transform_flow()`. This adds `eval()` and `eval_async()` methods to the function.
3. Call it directly inside your indexing flow definition — the body executes and the transformation is registered as part of the flow.
4. At query time, call `eval()` (or `eval_async()`) with concrete input data to run the transformation directly.

```python
@cocoindex.transform_flow()
def text_to_embedding(text: cocoindex.DataSlice[str]) -> cocoindex.DataSlice[NDArray[np.float32]]:
    return text.transform(
        cocoindex.functions.SentenceTransformerEmbed(
            model="sentence-transformers/all-MiniLM-L6-v2"))
```

### Using a transform flow in your indexing definition

Call the function directly inside your flow:

```python
with doc["chunks"].row() as chunk:
    chunk["embedding"] = text_to_embedding(chunk["text"])
```

You can also use the `call()` method to keep operations chainable:

```python
with doc["chunks"].row() as chunk:
    chunk["embedding"] = chunk["text"].call(text_to_embedding)
```

### Evaluating at query time

Call `eval()` with a concrete value — it returns the output type directly:

```python
print(text_to_embedding.eval("Hello, world!"))
```

In an async context, use `eval_async()` instead:

```python
print(await text_to_embedding.eval_async("Hello, world!"))
```

## Query handlers

Query handlers expose a simple function — taking a query string and returning structured results — that tools like CocoInsight can discover and invoke without additional glue code.

- **Define**: write a plain Python function `def search(query: str) -> cocoindex.QueryOutput`.
- **Register**: decorate with `@<your_flow>.query_handler(...)`.
- **Return**: a `cocoindex.QueryOutput(results=[...])`.
- **Optional metadata**: `QueryHandlerResultFields` tells tools which fields hold the embedding vector and score.

<Note>
The handler can be sync or async. The decorator registers the handler but does not change the function signature — you can still call the function directly.
</Note>

### Minimum query handler

```python
@my_flow.query_handler(name="run_query")  # name is optional; defaults to function name
def run_query(query: str) -> cocoindex.QueryOutput:
    # 1) Perform your query against the input `query`
    ...

    # 2) Return structured results
    return cocoindex.QueryOutput(results=[{"filename": "...", "text": "..."}])
```

`QueryOutput.results` is a list of dicts (or dataclass instances). All JSON-serializable types are supported. Embeddings can be `list[float]` or a NumPy array.

### Query handler with embedding and score metadata

Provide `result_fields` to help CocoInsight understand the structure of your results:

```python
@my_flow.query_handler(
    name="run_query",
    result_fields=cocoindex.QueryHandlerResultFields(
        embedding=["embedding"],  # path to the vector field in each result
        score="score",            # numeric similarity score (higher is better)
    )
)
def run_query(query: str) -> cocoindex.QueryOutput:
    # 1) Compute embedding for the input query (often via a transform flow)
    query_vector = text_to_embedding.eval(query)

    # 2) Run your database/vector store query
    ...

    # 3) Return structured results plus optional query_info
    return cocoindex.QueryOutput(
        results=[{"text": "...", "embedding": some_vec, "score": 0.92}],
        query_info=cocoindex.QueryInfo(
            embedding=query_vector,
            similarity_metric=cocoindex.VectorSimilarityMetric.COSINE_SIMILARITY,
        ),
    )
```

**`result_fields`** fields (all optional):

- `embedding` — list of keys navigating to the embedding in each result. Use multiple entries when you have multiple embeddings from different models.
- `score` — key pointing to a numeric field where larger means more relevant.

**`QueryOutput.query_info`** fields (all optional):

- `embedding` — the computed embedding for the query itself.
- `similarity_metric` — the similarity metric used when querying the index.

### Register without a decorator

You can register a handler imperatively using `add_query_handler()`:

```python
def my_search(query: str) -> cocoindex.QueryOutput:
    ...

my_flow.add_query_handler(
    name="run_query",
    handler=my_search,
    result_fields=cocoindex.QueryHandlerResultFields(embedding=["embedding"], score="score"),
)
```

## Resolving default target names

When you export data to a target without specifying an explicit name, CocoIndex generates a default name. Use `cocoindex.utils.get_target_default_name()` to resolve that name at query time.

| Argument | Type | Description |
|---|---|---|
| `flow` | `cocoindex.Flow` | The flow that owns the export target. |
| `target_name` | `str` | The export target name used in the `export()` call. |

```python
table_name = cocoindex.utils.get_target_default_name(text_embedding_flow, "doc_embeddings")
query = f"SELECT filename, text FROM {table_name} ORDER BY embedding <=> %s DESC LIMIT 5"
...
```
