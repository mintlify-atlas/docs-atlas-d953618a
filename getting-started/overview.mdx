CocoIndex takes data freshness to the next level. Its core value is **incremental processing** — only recomputing what has changed rather than reprocessing everything from scratch on every update.

The core engine is written in Rust with Python bindings via PyO3, giving you production-ready performance from day one.

## Programming model

CocoIndex follows the [Dataflow programming](https://en.wikipedia.org/wiki/Dataflow_programming) model. Each transformation creates a new field solely based on input fields, without hidden states or value mutation. All data before and after each transformation is observable, with lineage available out of the box.

Developers declare transformations rather than explicitly managing creates, updates, and deletes. CocoIndex handles keeping the derived index in sync with the source.

```python
# import
data['content'] = flow_builder.add_source(...)

# transform
data['out'] = data['content']
    .transform(...)
    .transform(...)

# collect data
collector.collect(...)

# export to db, vector db, graph db ...
collector.export(...)
```
